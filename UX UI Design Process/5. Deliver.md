## 5. 산출 단계

IA, 이용흐름, 인터랙션, UI, GUI 등 UX/UI 디자인 각 역영을 상세하게 설계하는 단계로, 이전 단계에서 만들어진 사용자 시나리오가 설계의 기반이 된다.

<br/>

### 5-1. IA 설계

IA는 UX/UI 디자인 설계의 큰 그림을 잡는 기초 설계 작업이다. IA가 설계돼야 이용흐름, 인터랙션 등이 설계될 수 있다. IA는 UX 여정 중 탐색에 가장 연관된 작업으로 메뉴, 검색, 필터/태그, 레이블링 등의 설계가 이루어진다.

#### IA 설계 순서

1. 정보 정의 : 서비스가 제공할 정보를 정의한다

   정보 계층구조, 유형, 현황, 정보 단위, 구성, 출처, 디지털 가공 여부, 제공 방식, 담당자, 업데이트 주기, 자용자 참여 방식 등이 정의서에 포함된다.

2. 정보구조 설계 : 정보의 계층구조를 만든다.

   한 단계의 하위 메뉴를 1 depth라고 하는데 모바일에서는 단계가 너무 깊은 설계를 지양한다. 또한 사용자 시나리오가 정보구조 상에서 실제로 잘 작동하는지 수시로 점검이 필요하다.

- 대표적인 4가지 정보구조
  1) Hierarchy : 하위 메뉴로 범위를 좁혀가며 탐색하는 구조로, 정보 규모가 크고 단계적으로 정보를 탐색하는 경험이 많은 서비스에 사용된다. 예) 대부분의 PC 상의 정보구조
  2) Hub&Spoke : 홈 화면이 허브 역할을 하며 개별 정보를 갔다가 홈으로 돌아오는 방식인데, 정보 규모가 작고 홈 화면 의존도가 높은 서비스에서 사용된다. 예) 넷플릭스, 유투브, 틱톡 등의 미디어 서비스
  3) nested doll(중첩인형) : 메인에서 탐색, 조회, 활동을 순차적으로 진행하고, 중간에 이전 단계로 돌아갈 수 있다. 정보 규모가 작으면서 이용흐름이 단선적인 서비스에 사용된다. 예) 날씨, 일정, 지도 등 소규모 서비스나 단일 상품 예약 서비스(항공, 숙박)
  4) Dashboard : Hub&Spoke와 유사하며, 추가적으로 홈 화면에서 요약 정보를 제공한다. 예) 헬스케어, 파일 관리, 스케줄, 핀테크 등

- 정보 분류 체계
  1) Top-down

  - 서비스에서 사용사가 행하는 업무가 기준 예) 결제, 주문, 조회, 문의
  - 정보의 성격 및 주제가 기준 예) 사회, 정치, 경제, 스포츠

  - 정보의 형식적인 속성이 기준 예) 최신뉴스, 인기/추천글, 커뮤니티
  - 사용자 유형이 기준 예) 아동청소년, 청년, 중장년

  2) Bottom-up

  - 메타데이터 선택에 기반
  - 사용자들이 만든 태그 기반
  - Q&A 기반

3. 내비게이션 설계

   내비게이션은 메뉴를 탐색할 때 사용하는 도구로 PC에서는 상단 글로벌 내비게이션(GNB)과 좌측 로컬 내비게이션(LNB)를 위주로 구성되고, 모바일에서는 PC와 유사하면서도 하단의 탭 메뉴, 개인화 메뉴 등이 추가로 구성된다. UI에서 내비게이션의 비중이 큰 PC와는 다르게 모바일에서는 스와이프, 롱탭 등 제스처를 통해 메인 UI에 감춰져 있는 내비게이션을 불러오는 형태가 많다.

4. 검색 설계

   키워드를 직접 입력하거나 조건을 구체적으로 선택해서 목표 정보에 빠르게 접근하는 과정으로, 검색 관련 설계는 검색 입력창 설계 외에도 옵션, 검색 결과 화면 구성, 연관 검색어, 추천 검색 결과 등에 대해 고려가 필요하다.

- 검색 설계 항목
  1) 검색어 입력 : 검색 입력창 위치, 자동 완성, 자동 추천, 검색 기록
  2) 검색 옵력 : 검색 카테고리 선택, 상세 검색 옵션, 검색 결과 필터링
  3) 검색 결과 : 연관 검색, 추천 검색 결과, 검색 결과 카테고리화, 검색 결과 정렬
  4) 검색 리소스 : 소셜 검색, 외부 리소스 검색, 메타데이터/시소러스

5. 필터/태그 설계

   특정 정보 범위나 단일 정보에 접근할 수 있게 돕는 기능이다. 필터는 원하는 조건을 사용자가 직접 적용해야 한다는 점에서 목적의식적 동기에서 이루어지고, 태그는 제시된 옵션을 선택하기만 하면 된다는 점에서 비목적의식적 동기로 이루어지는 편이다.

   설계 시에 개인화에 대한 고려도 필요한데, 사용자가 자주 찾는 조건들은 기본으로 필터가 세팅돼 있거나 관심사나 이전 서비스 사용 이력을 반영하여 태그를 제공할 수 있다.

6. 레이블링 설계

   서비스의 메뉴명, 정보명, 검색 조건명, 필터명, 태그명 등을 설계하는 작업으로, 레이블만 보아도 정보가 연상될 수 있게 각 정보를 대표하는 레이블을 정해야 한다.

<br/>

### 5-2. 이용흐름(User Flow) 설계

사용자가 특정 작업을 하기 위해 서비스에서 이동해가는 경로를 정리하는 것으로 사용자 시나리오보다 더 구체적으로 표현된다.

1. 이용흐름의 구성요소

   1) 목표(User goal) : 사용자가 서비스를 통해 얻고자 하는 결과
   2) 작업(Task) : 목표를 위해 사용자가 취한 방법
   3) UI : 서비스가 사용자에게 정보를 제시하기 위한 접점
   4) 행동 : 사용자가 작업을 완료하기 위해 UI 상에서 보이는 활동
   5) 판단 : 사용자 행동의 결과가 나누어 지는 것 예) 결과 없음, 결과 표시

2. 이용흐름 설계 과정

   작업 흐름 설계가 사실상 제일 중요하다.

   1) 사용자 시나리오에서 목표와 작업을 추출한다.
   2) **작업 흐름(Task flow)**을 설계한다.
   3) 화면 흐름(Wire flow)을 설계한다.
   4) UI 설계 과정에서 최종 이용 흐름을 완성하며, 제스처나 상태변화 등 인터랙션을 반영한다.

<br/>

### 5-3. 인터랙션 설계

UI를 생동감 있게 해주고, 이용흐름을 매끄럽게 만들어주는 역할을 하는 인터랙션에 대한 중요성은 구글의 Material Design을 기점으로 주목받기 시작했다.

> Material Design : 플랫 디자인의 장점을 살리고 빛에 따른 종이의 그림자 효고를 이용해 입체감을 주는 디자인 방식이다.  2014년 구글이 안드로이드폰에 적용하기 시작했다.

#### UX/UI 디자인에서 인터랙션 구성요소

1. 제스처(Gesture)

   가장 기본적인 인터랙션 요소로, 사용자가 취한 제스처와 기대된 결과가 일치하도록 맥락을 고려해 설계하는 것이 중요하다.

- 대표적인 제스처들
  1) 탭 : 화면 일정 부분을 눌렀다 뗀다.
  2) 더블 탭 : 화면 같은 부분을 일정시간 내에 두 번 눌렀다가 뗀다.
  3) 롱 탭 : 화면 같은 부분을 일정시간 이상 누른다.
  4) 스크롤 : 화면을 위/아래로 움직인다.
  5) 패닝 : 화면 한 지점을 누른 상태에서 움직인다.
  6) 스와이프 : 화면의 두 지점을 옆으로 이동했다가 뗀다.
  7) 드래그&드롭 : 화면의 한 UI 요소를 누른 채 이동했다가 뗀다.
  8) 핀치 인/핀치 아웃 : 떨어진 두 손가락을 누른 채 넓게/좁게 이동한다.

2. 선택/실행

   제스처를 통해 불러온 기능을 선택하고 실행해 화면 간을 이어주거나 화면 상태를 변화시키는 것이다.

- 모바일 선택/실행 방식
  1) 맥락 도구(Contextual tool) : 롱 탭을 하면 나타난다. 여러 항목에 대한 도구라면 상하단에, 특정한 건에 대해서만 실행되는 도구라면 선택한 항목 위에 나타나는 것이 좋다. 예) 카카오 채팅창 롱탭 시 채팅방 이름 설정, 나가기 기능
  2) 목록 내 활동(In-line Action) : 탭을 하면 바로 실행된다. 화면 목록 옆에 직접 기능이 표시되거나 제스처에 따라 숨겨져 있던 기능이 나타난다. 예) 뮤직차트 플레이 버튼, 구글 메일 작은 스와이프 했을 때 삭제, 전달 기능
  3) 툴바(Tool bar) : UI상에 고정돼 화면 하단 바 형태나 상단 (+) 아이콘 형태로 있다.
  4) 옵션 메뉴(Option Menu) : 큰텐츠 화면의 아무 곳이나 탭을 했을 때 도구가 나타난다. 예) 독서, 웹툰 앱
  5) 주활동 버튼(Call to Action Button) : 주활동을 마무리, 완료하는 중요한 버튼으로, 다른 버튼보다 강조된다. 예) 회원가입 버튼
  6) 상태변화 버튼(Multi-state Button) : 하나의 버튼으로 상태 변화 과정을 알려준다. 예) Remove 버튼을 한번 탭하면, Tab to delete 라는 버튼으로 바뀌고, 한번 더 탭하면 목록에서 제거되는 기능

3. 상태

   서비스의 현재 상태를 알리고, 사용자의 행동에 따라 피드백을 준다.

4. 상태 변화

   최근 모바일 트렌드를 이끄는 주요 인터랙션 구성 요소이다.

- 모바일에서 상태변화 방식
  1) 빠른 변화(Quick change)
  2) 뒤집기(Flip) : A화면이 뒤집어지는 효과와 함께 B화면으로 이동한다.
  3) 확장하기(Expand) : 동일 화면 내에서 요약돼있던 컨텐츠가 확장되면서 숨겨진 내용을 보여준다.
  4) 전체화면으로 열리기(Open to Full Screen) : 화면 내에 한 UI 요소가 확장되면서 전체 화면이 완전히 대체된다.
  5) 수평적 움직임(Horizontal slide) : 큰 스와이프를 했을 때 A화면과 B화면이 부드럽게 전환된다.
  6) 변환하기(Transformation) : 화면 내 UI 요소가 제스처에 따라 형태, 크기, 위치를 변화시키면서 다른 것으로 대체된다.

###	 5-4. UI 프로토타이핑

앞서 작업된 IA, 이용흐름, 인터랙션 설계가 종합되는 단계다. 아직 작업흐름(Task Flow)였던 이용흐름은 UI가 결합되어 완전한 이용흐름이 되며, 인터랙션 설계에서 정한 제스처, 선택/실행, 상태, 상태변화도 결합된다.

- UI 설계요소

  1) IA 설계에서 가져오는 요소 : 내비게이션, 레이블, 사용자-시스템, 사용자-사용자 간 커뮤니케이션 요소
  2) 인터랙션 설계에서 가져오는 요소 : 버튼, 기능, 링크, 배너, 도움말, 안내 등
  3) GUI 디자인과 함께 작업하는 요소 : 서비스 감성 전달을 위한 컬러, 이미지, 일러스트 같은 그래픽 요소, 아이콘, 불릿 같은 메타포, 로고, 아이덴티티 등

레이아웃 설계는 서비스 주요 화면을 설계하는 기본 작업으로, 큰 그림에서 시작해서 상세한 작업으로 들어가는 것이 화면 구성의 일관성을 놓치지 않는 방법이다. 주요 화면 구성을 그린 다음에는 정보나 기능을 어떻게 배치할 것인지 내비게이션과 인터랙션 방식을 고려해서 설계한다. 이 과정을 마친 결과물을 와이어 프레임이라고 하는데, 여기서 이미 좋은 UI인지 아닌지가 사실상 판가름 난다.

UI 화면 설계에 이르러서는 설계자 설명 없이도 내용이 전달될 수 있도록 실제 컨텐츠 혹은 그에 준하는 예시 컨텐츠가 담겨야 한다.

UI 화면 설계가 끝나면 작업 흐름대로 화면 내 상태변화와 화면 간 이동을 표시해 이용 흐름을 설계하며, 이 결과는 GUI 디자인, UI 개발 단계에서 지속적으로 활용된다.

UI 프로토타이핑은 테스트를 거친 후 UX 가이드라인과 UI 설계서로 이어지게 된다.

<br/>

### 5-5. GUI 프로토타이핑

UI 프로토타이핑과 동 시점에 진행되는 작업이다. 서비스 컨셉, IA, 이용흐름, 인터랙션을 기반으로 디자이너들은 서비스의 톤앤매너를 구상하고, UI 프로토타입을 받아 GUI 프로토타이핑 작업에 들어간다.

UI가 기능적이고 논리적인 영역이라면 GUI는 감각과 무의식의 영역이다.

1. 인지 이론

- 게슈탈트 원리
  1) 근접성(Proximity) : 거리상 가까운 요소와 떨어진 요소를 구분하는 경향 예) 가까이 위치한 항목끼리는 연관성이 있다고 생각한다.
  2) 유사성(Similarity) : 색상, 모양, 크기가 동일한 요소를 하나로 지각하는 경향 예) 유사한 형태가 나열돼 있으면 한 묶음으로 인식한다.
  3) 전경/배경(Figure Ground) : 전면에 드러난 개체와 후면의 배경에 동시에 주목하는 경향 예) 전경과 배경의 대비가 뚜렷한 것에 주목한다.
  4) 연속성(Continuation) : 어떤 선이나 운동 방향이 있을 때 그 윤곽을 먼저 인지하는 경향 예) 화면 구성이 3열 분할로 구성돼 있을 때 이 열을 먼저 주목한다.
  5) 폐쇄성(Closure) : 여러 요소가 단일 패턴을 보이면 그것을 하나로 인지하는 경향
  6) 공동 운명(Common Fate) : 개별 요소가 동일 방향으로 움직이면 그것을 하나로 인지하는 경향
- 인지 우열 관계
  컬러 > 간격 > 모양 > 크기
- PC와 모바일의 시선 흐름 비교
  PC는 스크린의 왼쪽 상단에서 시선이 시작하여 오른쪽으로 향하는 F형, Z형 흐름이 일반적이고, 모바일은 아래에서 위로 단순하게 시선이 이동한다.

2. 정보 구현

   1) 구분 : 가장 기본적인 방법은 간격이다. 정보의 거리가 떨어져 있으면 근접성 원리에 의해 사용자가 바로 인지하게 된다. 간격만으로 구분이 불명확한 경우 구분선을 사용할 수 있다. 이보다 더 강한 구분은 면으로 할 수 있는데 이는 UX 맥락에 따라 활용하는 게 좋다.
   2) 대비 : 복수의 정보나 기능 요소를 사용자들에게 '다르다'고 명확하게 인식시킬 필요가 있을 때 극적인 대비를 이루는 컬러를 활용할 수 있다.
   3) 강조 : 주변과는 확연하게 다른 디자인 언어를 사용해서 시각적인 효과를 과장하는 기법으로 여백, 크기, 배경 등을 활동할 수 있는데, 모바일의 좁은 화면에서는 보통 컬러를 활용한다.
   4) 리듬 : 동일한 디자인 요소들을 서비스에 일관되게 적용하여 시각적인 연결고리를 제공하는 것이다. 최근에는 상태변화를 통해 리듬감을 부여하는 것이 트렌드다.

### 5-6. UX 테스트

- UX 테스트의 종류

  원격 테스트나 AB 테스트보다는 사용성 테스트가 보편적이다.
  1) 사용성 테스트 : UXer들이 작업 수행 시간, 이동단계, 에러율, 학습 소요 시간 등 정량적인 지표를 측정해서 서비스의 문제를 진단한다.
  2) 원격 테스트 : 원격 테스트 툴을 사용해서 프로토타입을 검증받는다.
  3) AB 테스트 : 기존 서비스와 새로운 프로토타입을 A/B안으로 사용자들에게 어떤 것이 더 나은지 평가받는다.

<br/>

### 5-7. UX 가이드 라인

테스트를 통해 프로토타입 검증까지 마쳤으면, UI 기획자, GUI 디자이너, UI 개발자들이 후속 업무를 진행할 수 있도록 그동안의 결과물을 정리하는 작업이다.
